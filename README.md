# Polygon Zone

This custom Home Assistant integration lets you define complex geographic zones based on polygons from a GeoJSON file. The integration creates binary sensors that indicate whether a tracked device is inside or outside each polygon. Optional tolerance zones can be generated by offsetting the polygon.

## Installation

1. Copy the `poly_zone` directory into your Home Assistant `custom_components` folder.
2. Restart Home Assistant.

## Configuration

Use the Home Assistant UI to add the integration:

1. Navigate to **Settings → Devices & Services** and click **Add Integration**.
2. Search for **Polygon Zone**.
3. Provide:
   - A name for the zone group.
   - Path to a GeoJSON file containing `Polygon` or `MultiPolygon` features.
   - A device tracker entity to monitor.
   - Optional tolerance distance in meters for an expanded zone.

Options such as tolerance, invert logic, and automatic reload when the GeoJSON file changes can be adjusted in the integration's options.

## Events

The integration fires events when the tracked device enters or exits a zone:

- `poly_zone_enter`
- `poly_zone_exit`

Event data includes the device tracker entity, zone entity, and the device's current coordinates.

## Development

This repository contains the source for the integration. Run `python -m py_compile` and `pytest` to ensure code quality. The code uses a lightweight file watcher to reload zones when the GeoJSON file changes.
